## Transit Scheme Builder — New Project

### Context
Ignat has a side project: redesigning Minsk public transit bus stop info posters. Current Minsk posters are ugly — too many colors, inconsistent layouts, tiny unreadable stop names in circles, no standard composition.

### Design Direction (from Ignat)
- **London TfL spider map approach** — all routes from one focal stop, branching tree
- **Two-color system:** yellow (#F5A623) for buses, teal (#2D9B8A) for trolleybuses (Minsk standard)
- **References studied:** London spider maps, Moscow info posters, Berlin BVG route strips (old + modern)
- Consistent layout rules across ALL stops (the main win)
- Horizontal readable stop names, not tiny text in circles
- Metro transfers shown with red M icon
- Terminal stops with route badges

### Features
- Upload CSV/JSON route data or load sample (fictional Minsk stops)
- Select focal stop → auto-generates spider map
- Vertical trunk layout with horizontal branches (L-shaped connectors)
- Trie-based shared segment merging
- Two-color lines (parallel lines for mixed bus+trolleybus segments)
- Route badges at top and branch points
- Terminal badges at end of branches
- Metro M icons, "Вы тут" badge
- Toggle routes on/off, adjust spacing/column width/font/line width
- Pan/zoom canvas, export SVG/PNG

### Key Technical Details
- Trie builder: `buildTrie()` — merges shared segments from focal stop
- Structure extractor: `extractStructure()` — converts trie to trunk+branches (trunk = path with most routes)
- Layout: `computeLayout()` — vertical trunk, branches assigned left/right (bus→left, trolley→right heuristic)
- Renderer: `renderSVG()` — draws edges with `makeEdgePath()` (straight) and `makeEdgePathMixed()` (parallel offset for mixed types)
- Corner paths use SVG quadratic bezier (Q command) for rounded turns
