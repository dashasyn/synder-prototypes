### **FAQ (5 questions)**

| Question | Answer |
|----------|--------|
| What are Smart Rules? | Smart Rules are automated IF/THEN conditions. For example: IF a transaction is from Amazon AND over $500, THEN categorize it as "High-Value Equipment." Rules run on every new transaction without you touching it. |
| When should I use a Smart Rule? | Use rules when you see patterns in your data that require the same action every time — like categorizing specific products, splitting fees by percentage, or flagging certain transaction types for review. |
| Do rules apply to existing transactions? | No. Smart Rules only apply to new transactions coming in after the rule is created. To fix historical data, use Manual Journals or re-import with updated mapping. |
| Can I have multiple rules? | Yes. You can create as many rules as your plan allows. Rules can even chain together — the output of one rule can trigger another condition. |
| What if a rule stops working? | Rules only fail if the condition logic breaks (like referencing a deleted product). Synder will flag inactive rules so you can fix or delete them. |
