Pressure-test before going to high-fi

  1. Route identity: if route 78 runs both day and night with different stops, is it one route shown twice or two routes? If one, we need a thin Route object schedules reference. If two, duplicate is enough.
  2. Drift surfacing: "Used in 4 schedules · 1 with overrides" indicator on Stations — does this go far enough? Should the manager see *which* schedules deviate?
  3. Per-route rules vs station rules: "Show destination on front screen for whole journey" lives on Templates. But it's a route-property, not a station-property. Modeled as rule applied to all stations of a route — does that read right?
  4. Schedule-level overrides: are they always drift, or are some legitimate (Night phrasing)? Maybe distinguish "override" from "variant for night context".
  5. Audio composer ergonomics: chip-based is fast but order-sensitive. Is right-to-left/left-to-right read order obvious enough for non-native English manager?
