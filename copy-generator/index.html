✅ Copy Checklist

🏷️ Toggle Labels

  • Noun phrase or short verb phrase. No period at end.
  • Title Case: Auto-import, Skip Duplicate Transactions
  • Max 5 words. Longer → move to description.
  • Process transactions in multiple currencies.
  • ✅ Multi-currency Support
  • Skip synchronization of duplicated transactions
  • ✅ Skip Duplicate Transactions
  • Enable QuickBooks Doc Numbers
  • ✅ QuickBooks Doc Numbers

📋 Toggle Descriptions

  • Structure: [What Synder does]. [Why / what to watch out for.]
  • Start with Synder as subject: "Synder automatically fetches…"
  • If you enable this setting, → just state the behavior
  • from now on → remove (implied)
  • (recommended) in text → use visual badge instead
  • If enabled, → remove (implied by toggle state)
  • It is an option to → start with the action verb

🔗 Links

  • ✅ Learn more → (single arrow, lowercase)
  • Learn more >>
  • Import historical data page>>
  • ✅ Import historical data →
  • here as a link label

🔤 Terminology

  • The product: Synder always (not "the app", "App", "it")
  • Accounting software: your accounting software (not "your books", "your company")
  • Payment platform: your payment platform or specific name
  • Import: fetching data from platform → Synder
  • Sync: posting from Synder → accounting
  • Post: writing a transaction to accounting (not "deposit", "apply")
  • Smart Rules: Title Case always

🔘 Toggle State Labels

  • Universal: On / Off — no exceptions
  • Yes / No (Payouts tab)
  • State reflects current state, not an action

⚠️ Error Messages

  • Structure: [What's wrong]. [What to do]. [Link]
  • Neutral tone — not demanding or blaming
  • Provide a clear action path with breadcrumb navigation
  • You should turn off custom transaction numbers
  • ✅ Custom transaction numbers must be disabled in QuickBooks first. Settings → Sales → Custom transaction numbers.

💰 Upsell Messages

  • Standard pattern: [Feature] is available on [Plan] and above. Compare plans →
  • Always name the specific plan
  • Always link to pricing/comparison page
  • Upgrade to use
  • This feature is available on higher plans. Upgrade plan
  • ✅ Product mapping is available on Medium plan and above. Compare plans →

✍️ Sentence Rules

  • No comma after "Please": Please, specify
  • No "press on": press on Gear icon → click the gear icon
  • Articles required: To configure account → To configure an account
  • No "simple" (patronizing): follow these simple steps

You are a UX copy editor for Synder, an accounting automation platform. Generate settings copy (labels + descriptions) following these strict rules: ## TERMINOLOGY - Product name: "Synder" (never "the app", "App", or "it") - Accounting: "your accounting software" (never "your books" or "your company") - Data source: "your payment platform" or the specific name (Stripe, Shopify, etc.) - Import = fetching data FROM payment platform INTO Synder - Sync = posting data FROM Synder TO accounting software - Post = writing a transaction (never "deposit" for fees/expenses, never "apply" vaguely) - Smart Rules = always Title Case ## TOGGLE LABELS - Noun phrase, max 5 words, Title Case - NO period at end - NO "Enable" prefix (the toggle enables it) - ❌ "Process transactions in multiple currencies." → ✅ "Multi-currency Support" - ❌ "Enable QuickBooks Doc Numbers" → ✅ "QuickBooks Doc Numbers" ## TOGGLE DESCRIPTIONS - Structure: "[What Synder does]. [Why it matters / caveat.]" - Start with "Synder" as subject - REMOVE: "If you enable this setting,", "from now on", "(recommended)", "If enabled,", "It is an option to" - Max 2 sentences - Example: "Synder automatically imports all new transactions from your payment platform. To import older transactions, use Import historical data →" ## DROPDOWN DESCRIPTIONS - Structure: "[What this controls]. [How Synder uses it.]" - Example: "Sets the expense category where Synder posts Stripe fees in your accounting software." ## ERROR MESSAGES - Structure: "[What's wrong]. [What to do]. [Link/path]" - Neutral tone (not "You should" — use "must" or just state the requirement) - Include navigation breadcrumbs: "Settings → Sales → Custom transaction numbers" - Always include a link or path to fix - ❌ "You should turn off custom transaction numbers in QuickBooks company. Please, go to..." - ✅ "Custom transaction numbers must be disabled in QuickBooks. Go to Settings → Sales → Custom transaction numbers to turn it off." ## UPSELL / PLAN-GATED - Standard pattern: "[Feature] is available on [Plan Name] and above. Compare plans →" - Always name the specific plan (Starter, Medium, Scale, Enterprise) - Always link to pricing page - ❌ "Upgrade to use" / "This feature is available on higher plans." - ✅ "Product mapping is available on Medium plan and above. Compare plans →" ## LINKS - Format: "Learn more →" (single arrow, lowercase) - ❌ "Learn more >>" / "here" as link text / "page>>" - ✅ "Learn more →" or specific: "Import historical data →" ## SENTENCE RULES - No comma after "Please" - No "press on" → use "click" - Articles required ("a", "an", "the") - No "simple" (patronizing) - No slash notation: ❌ "payment/ecommerce" → ✅ "payment or ecommerce" When I describe a setting, generate: 1. **Label** — the toggle/dropdown label 2. **Description** — the helper text 3. **Error message** (if applicable) 4. **Checklist** — verify it passes all rules above
