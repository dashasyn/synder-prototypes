🚨 Top 3 Critical Issues
🔥 On/Off vs Yes/No

Payouts tab uses Yes/No toggles while every other tab uses On/Off. One page, two toggle paradigms. Payouts is the 6th most-visited tab — thousands of users hit this inconsistency every month.

🔥 Stacking toasts

Toggling a single switch triggers 3 stacked toast notifications: "Success. Settings of organization successfully updated." shown three times simultaneously. Severe feedback noise — breaks trust and readability.

🔥 Recommended = Off

Auto-import and Auto-sync say "(recommended)" in their descriptions but ship as Off by default. Signals broken defaults at the exact moment users are configuring for the first time.

📐 Copy Strategy
A unified system. Apply to all 11 tabs. Once consistent, users build mental models faster.
💡
Core principle: Every description answers: What does Synder do when this is on? Why does it matter? — max 2 sentences. No preambles. No "from now on."

🏷️ Toggle Labels

  • Noun phrase or short verb phrase. No period at end.
  • Sentence case (capitalize first word only): Auto-import, Skip duplicate transactions
  • Max 5 words. Longer belongs in the description.
  • ❌ Process transactions in multiple currencies.
  • ✅ Multi-currency support
  • ❌ Archive Pending transactions after set number of days
  • ✅ Auto-archive pending transactions
  • ❌ Skip synchronization of duplicated transactions
  • ✅ Skip duplicate transactions

📋 Toggle Descriptions

  • Positive framing always: describe what happens when ON.
  • Structure: [What Synder does]. [Why it matters or what to watch out for.]
  • ❌ Remove: If you enable this setting, / If enabled,
  • ❌ Remove: from now on
  • ❌ Remove: (recommended) from body — use a visual badge instead
  • ✅ Start with Synder as subject: Synder automatically fetches…
  • For settings that disable something: When off, Synder skips…

🔗 Links

  • Always: Learn more → (single arrow, lowercase)
  • Internal nav: Import historical data →
  • ❌ Import historical data page>>
  • ❌ Smart rules >>
  • ❌ Learn more >>
  • ❌ here as a link label (accessibility anti-pattern)
  • ✅ Learn how to enable multicurrency in QuickBooks →

🔤 Terminology — Lock These Down

  • Synder: Synder always, capitalized (not "the app", "App", "it", "the system")
  • Books: your books (generic term for any accounting company — not "your company", "your accounting")
  • Integration: your integration or the specific name (Stripe, Shopify)
  • Import: fetching data from integration into Synder
  • Sync (Synchronization): synchronizing data from Synder to books
  • Sync: the main action — synchronizing transactions to books
  • Smart Rules: Title Case, always (not "Smart rules", "smart rules")
  • Enable / Disable: standard verbs for toggling features (not "turn on", "turn off", "switch on/off")
  • Click: for UI elements (not "press", "tap")
  • Organization: main unit in Synder (has accounting company + integrations + subscription)

🔘 Toggle State Labels

  • Universal standard: On / Off — no exceptions
  • ❌ Yes / No (Payouts tab currently — fix this)
  • The toggle state should reflect the current state, not an action
  • "On" = feature is active, "Off" = feature is inactive

🍞 Toast Messages

  • Fire once per save action. Current behavior: fires 3×.
  • ✅ Success/enable/change: {Thing} {past participle}! — exclamation mark
  • ✅ Settings updated! · Location tracking enabled!
  • ✅ Disable: {Thing} disabled. — period, neutral tone
  • ✅ Auto-import disabled.
  • ❌ Error: Couldn't {verb} {noun}. — period, calm tone
  • ❌ Couldn't save settings.
  • 🚫 No "was successfully" — ever. It's filler.
  • 🚫 Success. Settings of organization successfully updated. — redundant, too long

✍️ Sentence-Level Rules

  • No comma after "Please": ❌ Please, specify a category → ✅ Specify a category
  • No "press on" for UI interactions: ❌ press on Gear icon → ✅ click the gear icon. Use "click" for all UI elements; reserve "press" for keyboard shortcuts only.
  • Articles required: ❌ To configure account → ✅ To configure an account
  • No "simple" (patronizing): ❌ follow these simple steps → ✅ follow these steps
  • No "It is an option to": ❌ It is an option to assign a location → ✅ Tags each transaction with a location
  • Footer: standardize Terms of Service (Title Case, matches Privacy Policy)

🔴 1. "Update unavailable" — Permanent Save Blocker
PopoverAll 11 tabs
"Update unavailable: wait until your settings page is loaded and try again."
Trigger

Clicking the Update button on any settings tab. The page embeds 29 hidden spinner elements (fa-spinner fa-spin) across form fields. A JS check blocks saves while any spinner exists in the DOM — but since the spinners' parent elements have class="hidden", they're invisible yet still counted. The loading state never resolves.

UX Impact
  • 100% save failure rate — users can change any setting but never save
  • Message implies user should "wait" — but there's nothing to wait for
  • No timeout, no retry, no help link, no progress indicator
  • Appears as a small Bootstrap popover — easy to miss entirely
  • Users may think their settings did save and leave confused when nothing changes
Current CopyOption A — MinimalOption B — Full Rework✅ Approved
Update unavailable: wait until your settings page is loaded and try again. Settings are still loading. Please wait a moment and try again. Refresh page Some settings haven't loaded yet. You can save what's ready now or refresh the page to reload everything.
Engineering fix: The JS check should count only visible spinners (check offsetParent !== null or parent visibility). The hidden spinners with class="hidden" parents should not block saves. Also add a 10-second timeout with a graceful fallback message.
