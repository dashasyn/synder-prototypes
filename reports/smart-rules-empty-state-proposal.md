### FAQ

**Q: What can Smart Rules do?**
They automatically handle repetitive categorization tasks. Examples: categorize all Amazon purchases over $500 as "Equipment," split Shopify fees by percentage, or flag international transactions for review. You define the condition (IF) and the action (THEN).

**Q: When should I create a rule?**
Create a rule when you find yourself manually fixing the same type of transaction repeatedly. If you keep re-categorizing "Office Supplies" from a specific vendor, that's a perfect candidate for automation.

**Q: Will rules change my past transactions?**
No. Smart Rules only apply to new transactions that come in after you create the rule. Your historical data stays exactly as it is. To fix old transactions, use Manual Journals or re-import with updated mapping.

**Q: Can I edit or delete a rule?**
Yes, anytime. You can pause, edit, or delete rules. If you delete a rule, it stops applying to future transactions — but it doesn't undo any actions it already took.

**Q: What if two rules conflict?**
Rules run in the order you set. If two rules could apply to the same transaction, the first one in your list wins. You can drag to reorder rules anytime.
