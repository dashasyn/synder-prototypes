# Brief: Transaction Reconciliation Empty State Engagement

## The Problem
Very few users click the "Start reconciling" button on the empty state. We need to understand why users don't click, and how to engage them.

## Timeline Context
- **March 30, 2026:** Better empty state introduced
- **Analysis period:** March 30 - April 16, 2026 (~18 days)

## Previous Research (March 2026)
From earlier LogRocket analysis:
- 501 sessions/mo visited Tx Reconciliation landing page
- 148 went to /create (29.5%)
- 85 completed to detail page (17%)
- **22.5% clicked "Start reconciling" CTA**
- 0 rage clicks on landing page (users not frustrated — they just don't convert)
- 0 sessions under 30s (users spend real time on page)

## Key Previous Findings
1. **Beta badge = financial data fear** — users see "Beta" and hesitate to touch financial data
2. **"Start matching" CTA sounds like immediate action, not config** — users think clicking it will immediately start processing, not open a setup form
3. **No reversibility signal** — users don't know if they can undo or change settings after starting

## Empty State Elements (from memory)
The empty state includes:
- Illustration (friendly, abstract)
- Headline: "Transaction reconciliation" or similar
- Subtitle explaining what it does
- **Primary CTA: "Start reconciling"** (blue button)
- Trust line (e.g., "Free to use" or similar)
- **6 FAQ items** including:
  - "What is transaction reconciliation?"
  - "How does it work?"
  - "How is this different from balance reconciliation?" (uses jargon "clearing accounts")
  - "No sync balance changes" (misleading — implies no risk)

## The Core Questions
1. Why don't users click "Start reconciling"?
2. Is it fear (Beta, financial data), confusion (what will happen?), or lack of interest (don't see value)?
3. What do users do instead? (Scroll? Click FAQs? Navigate away? Click sidebar?)
4. How can we increase engagement?

## Constraints
- Feature is **free** — but users don't seem to know this or care
- Users may not understand what "reconciliation" means
- "Beta" badge may be creating trust issues with financial data
- CTA language may imply irreversible action

## Persona Perspectives Needed
- **Dasha (UX researcher):** Friction points, cognitive load, behavioral analysis
- **Sarah (Accountant):** Professional perspective — what's missing for an accountant to feel confident starting?
- **Mike (Business owner):** Non-accountant perspective — why would a regular user hesitate? What would make them click?

## Deliverable
For each persona: analyze the empty state (based on description above and previous findings), identify why users don't click, and suggest specific improvements to increase "Start reconciling" engagement.
